### Data Pipeline Architecture (Knowledge Transfer)

Onboarded through a KT (Knowledge Transfer) session to understand the client's existing data ingestion pipeline, which follows this flow:

I worked with the client's existing data ingestion pipeline which used the following tools:

1. **Qlik Replicate** extracts data from source databases and lands files into a **Google Cloud Storage (GCS)** bucket. 

2. **Argo Workflows** runs on a cron schedule, picking up files from the GCS landing zone bucket and triggering the downstream processing.

3. **dbt (Data Build Tool)** handles the backend transformation logic, resulting in tables being created and maintained in **Google BigQuery (BQ)**.

**Role:** Key Responsible for frontend and backend development end-to-end.

**Tech Stack:**
